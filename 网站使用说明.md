## Docker环境配置

### 安装Docker Desktop（Windows）

1. 下载并安装 [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)
2. 启用 WSL 2 功能（推荐）
3. 重启计算机
4. 启动 Docker Desktop，确保状态为运行中（绿色图标）

## 本地运行网站

### 使用标准版镜像（约400MB）

```powershell
cd d:\TIMpage\Flyincodesky.github.io
docker compose pull
docker compose up
```

### 使用轻量版镜像（约100MB，推荐）

```powershell
cd d:\TIMpage\Flyincodesky.github.io
docker compose -f docker-compose-slim.yml pull
docker compose -f docker-compose-slim.yml up
```

### 访问网站

启动成功后，在浏览器访问：**http://localhost:8080**

### 停止网站

按 `Ctrl + C` 或运行：
```powershell
docker compose down
```

## 更新网站内容

### 1. 首页修改

**文件位置：** `_pages/about.md`

可修改内容：
- 实验室简介
- 导师信息
- 重点项目展示

### 2. 团队成员

**文件位置：** `_pages/team.md`

更新步骤：
1. 替换成员照片到 `assets/img/` 文件夹
2. 修改成员信息（姓名、研究方向、简介等）
3. 添加/删除团队成员卡片

### 3. 研究项目

**文件夹：** `_projects/`

每个项目一个独立的 `.md` 文件，包含：
- 项目标题、描述
- 项目信息（经费、周期等）
- 研究内容
- 主要成果
- 相关链接

**添加新项目：**
```markdown
---
layout: page
title: 项目名称
description: 项目简介
img: assets/img/项目图片.jpg
importance: 1
category: 在研项目
---

## 项目内容...
```

### 4. 论文成果

**文件位置：** `_bibliography/papers.bib`

**添加论文：**
```bibtex
@inproceedings{作者2026标题,
  abbr={会议简称},
  title={论文标题},
  author={作者1 and 作者2},
  booktitle={会议全称},
  year={2026},
  pdf={论文PDF文件名.pdf},
  code={https://github.com/...},
  selected={true},  # 是否在首页展示
  bibtex_show={true}
}
```

**PDF文件：** 放置在 `assets/pdf/` 文件夹

### 5. 新闻动态

**文件夹：** `_news/`

**添加新闻：**

创建文件 `_news/announcement_XX.md`：
```markdown
---
layout: post
date: 2026-01-19 10:00:00+0800
inline: true
related_posts: false
---

新闻内容简短描述...
```

### 6. 活动新闻

**文件位置：** `_pages/activities.md`

直接编辑添加活动条目即可。

### 7. 联系方式

**文件位置：** `_pages/contact.md`

更新：
- 地址信息
- 联系电话
- 电子邮箱
- 社交媒体链接

## 网站结构说明

```
Flyincodesky.github.io/
```

## 常见问题

### Q1: 如何更改网站标题？
**A:** 修改 `_config.yml` 中的 `title` 字段

### Q2: 如何添加新的导航菜单？
**A:** 在对应的 `.md` 文件中设置 `nav: true` 和 `nav_order: X`

### Q3: 图片显示不出来？
**A:** 
- 确认图片路径正确：`assets/img/文件名.jpg`
- 图片文件名不要包含中文和空格
- 使用相对路径

### Q4: 修改后网站没有更新？
**A:** 
- Docker容器支持热重载，稍等片刻即可
- 如果还是不行，重启容器：
  ```powershell
  docker compose down
  docker compose up --build
  ```

### Q5: 如何部署到GitHub Pages？
**A:** 
1. 将代码推送到GitHub仓库
2. 在仓库Settings -> Pages中配置部署分支为 `gh-pages`
3. GitHub Actions会自动构建和部署

### Q6: 搜索功能如何使用？
**A:** 搜索功能已在 `_config.yml` 中启用：
- `search_enabled: true` - 启用全局搜索
- `posts_in_search: true` - 搜索博客文章
- `bib_search: true` - 搜索论文

点击导航栏的搜索图标即可使用。
